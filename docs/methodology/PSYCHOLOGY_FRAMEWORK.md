# 心理学分析框架

PsyLens 将社区反馈拆成两个相互配合的观察层：

1. **表层话题**：玩家正在讨论什么问题，例如平衡、匹配、奖励、活动设计或社区冲突。
2. **体验机制**：这段表达呈现了怎样的心理体验，例如胜任受挫、公平受损、信任落差或归属感下降。

这套设计用于组织反馈与形成可检验的产品假设。机制标签承担解释线索的作用，实践落地时仍需进行心理测量、访谈诊断或因果推断。

## 1. 方法取向

项目采用“构念驱动编码 + 主题分析”的组合思路：

- 先依据心理学与用户体验研究中的成熟构念，预先定义有限的机制类别；
- 再保留表层话题编码，让具体问题不被抽象构念覆盖；
- 同时设置 `other_uncertain` 与 `uncertain`，把信息不足显式保留下来；
- 通过“话题 × 机制”的交叉分布寻找更具体的产品线索。

这一做法接近演绎式主题分析：研究问题与理论框架决定初始编码维度，实际文本仍决定证据是否能够进入某个类别。

## 2. 核心构念

| 机制 | 理论线索 | 在社区反馈中的操作性含义 | 常见表达 |
| --- | --- | --- | --- |
| 胜任受挫 `competence_frustration` | 自我决定理论中的胜任需要 | 玩家感到投入难以转化为有效表现、任务完成或可理解的进步 | “做不到”“白练”“选了也发挥不了作用” |
| 公平受损 `fairness_threat` | 公平判断与组织公正研究 | 玩家把负面结果归因于规则、匹配、资源分配或机制偏向 | “系统在控胜率”“分配明显不合理” |
| 信任与沟通落差 `trust_communication_gap` | 组织信任与程序透明 | 玩家认为说明不足、反馈无回应、承诺与执行不一致 | “改动没有说明”“反馈长期无回应” |
| 归属感下降 `belonging_drop` | 社群归属与心理共同体感 | 玩家与游戏、社群或身份群体之间出现疏离 | “环境已经变了”“不再想参与” |
| 规范与安全风险 `norm_safety_risk` | 群体规范、秩序与心理安全 | 反馈涉及外挂、骚扰、辱骂、误封、账号安全和举报机制 | “举报无效”“社区攻击性过强” |
| 不确定 `uncertain` | 测量中的保留类别 | 文本信息不足、依赖上下文或同时轻微指向多个机制 | 短句、纯语气、上下文缺失表达 |

### 胜任受挫的解释范围

胜任受挫关注玩家是否感到自己能够理解规则、有效行动并看到投入结果。在游戏语境中，它可能来自难度、数值、角色适配、任务结构或团队协作。该标签只描述文本中可观察的体验方向，不推断玩家稳定人格、真实能力或临床状态。

### 公平受损的解释范围

公平受损关注结果与程序是否被玩家体验为合理。匹配、数值、奖励和规则都可能触发公平判断。编码时需要区分“我做不到”和“系统分配不合理”：前者偏向胜任，后者偏向公平。

### 信任、归属与规范

信任落差用于描述玩家与运营方之间的沟通关系；归属感下降用于描述玩家与社群或游戏身份的疏离；规范与安全风险用于记录会直接影响参与安全和互动秩序的问题。三者在实际表达中可能同时出现，因此以证据单元的主要诉求为准。

## 3. 表层话题与机制的交叉

单独统计话题只能说明“问题集中在哪里”，单独统计机制只能说明“体验方向是什么”。两层交叉后，含义更清晰：

| 交叉组合 | 可形成的分析问题 |
| --- | --- |
| 平衡 × 胜任受挫 | 某些机制是否让玩家难以形成有效策略或发挥角色作用？ |
| 平衡 × 公平受损 | 数值与规则是否被体验为偏向特定角色、阵容或玩法？ |
| 匹配 × 公平受损 | 玩家是否把连胜连败、队友质量或对手分配归因于系统控制？ |
| 沟通透明 × 信任落差 | 公告、解释和反馈闭环是否足以支撑玩家理解改动？ |
| 社区冲突 × 规范安全 | 互动规则、举报和处置机制是否降低了参与安全感？ |

交叉组合只用于提出待验证问题。后续仍需结合访谈、问卷、行为日志、A/B 测试或上线指标判断影响范围与因果关系。

## 4. 编码顺序

1. 判断文本能否作为独立证据；
2. 确定主要表层话题；
3. 判断主要体验机制；
4. 记录最能支撑判断的原文片段；
5. 对信息不足的文本保留不确定标签；
6. 聚合时同时查看数量、平台、证据密度和编码来源。

## 5. 可靠性控制

- 每条证据优先使用一个主话题和一个主机制，降低口径漂移；
- 采用明确的纳入、排除和相邻标签区分规则；
- 保留不确定类别，避免为提高覆盖率而牵强归类；
- 统计时分开报告“机制不确定”和“证据纳入提醒”；
- 平台比较同时报告每样本证据密度，避免把文本长短与切分数量误当作意见强度；
- 后续双人复编码时，可计算 Krippendorff’s alpha 或其他适合名义变量的一致性指标。

## 6. 解释边界

当前公开编码来自两种历史生产方式：NGA 与贴吧主要保留历史 AI 编码，B 站使用离线规则提案。不同来源的编码策略并不完全等价，因此现有分布适合用于方法审计、探索性描述和后续研究设计。
