# 证据单元切分指南（EVIDENCE_UNIT_GUIDE）

> 定义「什么算一条证据单元」，以及如何从原始反馈 `raw_text` 中切分候选单元。用于 B 站候选切分与后续人工复核。

## 1. 什么是证据单元

证据单元是从一条原始反馈中切出的、**语义相对独立、可单独判断表层话题与机制**的最小文本片段。

- 一条 `raw_text` 可切出 0 到多个证据单元。
- 证据单元的 `unit_text` **必须逐字出现在**对应样本的 `raw_text` 中（可在归一化后定位）。
- 不修改原文：不改写、不纠错、不补全。

## 2. 切分规则（确定性）

当前 B 站候选切分（`tools/build_v2_dataset.py`）采用确定性标点切分：

- 切分符号：中文句号/问号/感叹号/分号（。！？；）、对应半角（.!?;）、换行。
- 切分后逐段 `strip()`。
- **最小长度 `min_len = 6`**：短于 6 个字符的片段不单独作为候选（视为语气词/连接词）。
- 同一样本内候选序号 `candidate_unit_index` 从 1 连续编号。

## 3. 是否作为正式证据（include_as_evidence）

候选单元进入离线规则基线提案后，判定是否可作为证据：

- `yes`：语义独立、可判断话题/机制、含明确体验或诉求。
- `no`：过短、纯语气词、纯上下文依赖、无独立含义、无情绪/诉求指向。
- `uncertain`：语义存疑、可能有信息但不足以判定。

## 4. 不得的做法

- 不得为追求覆盖率把无意义片段标为 `yes`。
- 不得把模型/Agent 置信度当作证据强度。
- 不得跨样本合并文本。
- 不得把「上下文才有意义」的片段单独当作独立证据（应标 `no` 或 `uncertain`）。

## 5. evidence_phrase

`evidence_phrase` 是证据单元中**最能支撑标签判断的关键短语**，必须能在 `candidate_unit_text` 中定位（归一化子串）。用于人工复核时快速核对判断依据。

## 6. 与机制/话题手册的关系

切分只解决「切出什么」；`MECHANISM_CODEBOOK.md` 与 `SURFACE_TOPIC_CODEBOOK.md` 解决「怎么打标签」。三者配合构成编码规范。
