# PsyLens 内容表达规范

## 1. 目标

PsyLens 的页面、README、方法文档、Prompt、CLI 输出、自动报告和正式项目说明统一围绕四个问题组织：

1. 当前提供什么能力；
2. 结果由哪些数据和步骤生成；
3. 这些结果适合支持哪些判断；
4. 使用者可以从哪里继续查看、运行或扩展。

内容应保持准确、简洁、可操作，并让读者先看到项目价值、方法和结果。

## 2. 内容分层

### A. 展示层

包括：

- `docs/index.html`；
- README 首屏与主要结果；
- 正式项目说明 DOCX；
- 页面加载的 JSON 文案。

展示层优先使用：

- 当前结果；
- 方法价值；
- 应用场景；
- 数据口径；
- 复现入口；
- 扩展入口与后续研究方向。

同一项适用范围集中说明一次，避免在多个板块重复状态提示。

### B. 方法与开发层

包括：

- `docs/methodology/**`；
- `docs/evaluation/**`；
- `docs/pipeline/**`；
- `pipeline/**`；
- Python 的模块说明、CLI 帮助和报告模板。

方法层使用中性、操作化表达：

- 描述当前模式和启用条件；
- 用覆盖率、状态和检查结果代替笼统评价；
- 将问题写成校准目标、诊断项或扩展任务；
- 将安全要求写成明确的配置方式。

### C. 证据与机器字段

包括：

- 公开样本与证据文本；
- 标签值；
- 失败类型 ID；
- 测试断言；
- 日志中的结构化错误码。

原始社区文本保持证据原貌；标签、错误码和断言保持机器可判定性。内容规范不用于改写证据文本，也不削弱数据安全与发布校验。

## 3. 推荐表达结构

### 结果说明

推荐：

> 486 条证据进入显式保留类别，构成标签边界校准的重点样本池。

避免把同一结果反复写成项目缺陷或泛化评价。

### 日期覆盖

推荐：

> 日期字段覆盖 120 / 360 条样本。当前版本聚焦横截面结构，时间维度可在日期覆盖扩展后启用。

### 编码来源

推荐：

> 当前数据保留历史模型编码与规则基线来源字段，平台结果按来源分层呈现；仓库同时提供多代理复检与共识工具，用于生成自动校准参考。

### 推断范围

推荐：

> 当前比例用于描述公开样本中的编码结构，并为访谈、问卷、行为分析和实验设计提供问题线索。

### 外部连接

推荐：

> 默认模式在本地确定性运行。外部模型与平台适配器通过显式 provider 和独立配置启用。

### 凭据与隐私

推荐：

> Cookie、Token、API Key 和登录态仅通过本地环境注入；仓库配置保存字段名、参数模板和脱敏示例。

## 4. 术语替换

| 原表达类型 | 统一表达 |
| --- | --- |
| 解释边界 | 结果使用说明 / 适用范围 |
| 质量风险 | 校准重点 / 关注项 |
| 缺失字段 | 字段覆盖情况 |
| 做不了 / 无法完成 | 当前模式 / 启用条件 / 扩展入口 |
| 未完成 | 当前版本聚焦于…… |
| 待复核状态 | 编码来源与校准状态，集中说明一次 |
| 不支持外部模型 | 外部模型通过 provider 扩展 |
| 不能用于因果 | 用于生成可验证假设，因果检验由实验模块承接 |
| 真实采集不可用 | 提供采集契约与配置模板，平台适配器按目标环境接入 |
| 失败 | 校验未通过 / 需要重新生成 / 进入诊断队列 |

## 5. 页面结构原则

页面优先顺序：

1. 项目目标与案例；
2. 关键结果；
3. 心理学与数据方法；
4. 可靠性评测；
5. 自动校准与争议分布；
6. 产品研究方向；
7. 结果使用说明；
8. Demo 与资料入口。

状态信息只在与读者决策直接相关时出现。内部任务状态、分支名、生产过程和返修记录进入 Issue、PR 或 artifact。

## 6. Python 与自动报告

Python 中的用户可见内容遵循以下规则：

- CLI 描述当前命令的输入、输出和默认模式；
- 成功输出包含产物路径、行数、哈希和状态；
- 校验结果使用稳定状态值，例如 `PASS`、`REVIEW`、`REBUILD`；
- 错误信息指出对象、期望值、实际值和恢复动作；
- 自动报告使用“结果、依据、建议动作”三段式；
- 注释与 docstring 说明设计目的和数据契约，避免自我评价式措辞。

## 7. 自动检查

`tools/lint_public_copy.py` 将扫描：

- `README.md`；
- `docs/**/*.md`；
- `docs/index.html`；
- `pipeline/**/*.md`；
- Python 中的 docstring、CLI 帮助、报告模板和固定展示文案。

检查项包括：

- 重复的状态提醒；
- “不是……而是……”模板句式；
- 以不足为中心的标题或卡片；
- 页面中的内部工程状态；
- 同一适用范围在多个板块重复出现；
- 缺少下一步入口的诊断文案。

原始数据文件、测试断言、错误码和安全关键字采用单独规则集。

## 8. 发布审查

每次页面或方法更新需要同时检查：

- 页面、README、DOCX 与 JSON 数字一致；
- 适用范围集中说明；
- 数据来源与编码来源准确；
- 结果陈述包含分母；
- 诊断项附带校准动作；
- 用户可以找到数据、方法、Demo 和完整项目说明；
- Ubuntu 与 Windows CI 通过。
