# 平台采集指南

真实平台访问默认不进入 CI。当前公开版本提供采集方法、字段契约和 dry-run 候选发现模板，没有把依赖旧页面结构的历史脚本标记为可直接运行工具。

## 1. 候选讨论登记

先使用 `pipeline/discovery/candidate_discovery_template.py` 生成候选表结构，或按同一字段人工登记：

- 平台；
- 检索主题；
- 时间窗；
- 内容入口；
- 主帖或回复类型；
- 采集优先级；
- 排除理由。

候选阶段应覆盖不同主题和互动强度，避免只保留热门或极端表达。

## 2. 真实采集需要实现的能力

- 明确 User-Agent；
- 请求限速与随机抖动；
- 分页终止条件；
- 最大重试与指数退避；
- HTTP 状态与解析失败日志；
- 原始响应缓存；
- checkpoint 与增量续跑；
- 平台 ID 或规范化文本去重。

## 3. 运行前提

- 遵守目标平台使用条款、访问限制和适用法律；
- 登录态和 Cookie 只在本地使用；
- 原始响应先写入本地缓存，不直接覆盖公开数据；
- 先用小样本验证页面解析、字段完整性和退出条件；
- 页面结构或接口发生变化时，重新验证解析规则。

## 4. 安全红线

- 不提交 Cookie、Token、账号密码、浏览器 profile 或登录缓存；
- 不在日志中打印凭据；
- 不把原始来源链接和账号定位字段写入公开数据；
- 不把历史页面选择器或接口参数描述为长期稳定能力。

## 5. 离线替代

只需体验分析链路时，运行：

```bash
python tools/run_demo.py --provider mock --output artifacts/demo/run
```

该命令不访问网络、不读取密钥，也不修改公开数据。
