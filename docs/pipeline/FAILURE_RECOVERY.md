# 失败与恢复

## 1. 候选发现与采集

- **访问受限或限速**：停止连续请求，记录失败时间和状态码，按指数退避重新执行；
- **登录状态失效**：在本地环境重新建立登录态，Cookie 和浏览器 profile 不进入仓库；
- **分页中断**：以候选清单和已完成页码作为 checkpoint，只补采失败部分；
- **页面结构变化**：保存失败页面样本，先更新解析规则并用小样本验证；
- **重复采集**：使用平台 ID、来源入口或规范化文本哈希去重。

## 2. 清洗与规范化

- **必填字段缺失**：阻止进入公开数据，修正字段映射后重新生成；
- **可选日期缺失**：保留为空并记录数量，时间分析排除或补充该部分；
- **公开字段出现 URL**：阻止发布，检查脱敏规则和字段白名单；
- **空表层话题**：统一转换为 `other_uncertain`；
- **重复文本组**：输出重复 ID，确认是重复抓取、引用复制或真实相同表达。

## 3. 证据与编码

- **父样本不存在**：证据不能发布，重新检查 ID 映射；
- **证据文本无法定位**：检查切分、空白归一化和文本改写，证据必须能够回到父样本；
- **非法标签**：根据 codebook 修正或使用显式不确定类别；
- **不确定比例异常升高**：分平台、标签来源和文本长度定位原因，检查构念边界与规则覆盖；
- **平台分布突变**：同时比较每样本证据密度和编码来源，避免把切分差异解释为意见变化。

## 4. 公开分析与文档

- **analysis_summary 过期**：重新运行 `tools/summarize_public_analysis.py`，提交生成结果；
- **页面数字与公开数据不一致**：以公开数据和生成器为准，禁止手工维护同一数字的多个副本；
- **比例缺少分母**：补充分子、分母与排除规则；
- **DOCX 生成失败**：先验证公开 CSV 与分析 JSON，再运行项目说明生成器；
- **页面静态加载失败**：页面保留静态回退值，同时检查 JSON 路径和浏览器控制台。

## 5. Demo 与 CI

- Demo 相同输入输出不一致时，检查时间戳、随机数、排序和平台换行符；
- manifest 哈希不一致时重新生成产物，不手工修改生成文件；
- Ubuntu 或 Windows 任一失败时，修正跨平台路径、编码或换行问题后重新运行；
- Pages 只部署通过公开数据、页面和文档校验的 `docs/`。

## 6. 输出隔离

运行脚本默认写入 `artifacts/`。只有经过校验的公开样本、证据、分析 JSON 与正式项目说明进入版本库。原始缓存、登录信息、抓取中间文件和外部模型密钥始终保留在本地环境。
