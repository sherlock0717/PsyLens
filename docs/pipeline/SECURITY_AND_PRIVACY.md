# 安全与隐私

## 1. 凭据安全

以下内容禁止提交到仓库：

- Cookie、Token、API Key 和账号密码；
- 浏览器 profile 与登录态缓存；
- 未审查的页面响应和抓取缓存；
- 包含凭据、请求头或本地路径的日志。

`.gitignore` 应覆盖 `.env`、Cookie 文件、缓存和 `artifacts/`。真实模型与真实采集只在用户明确配置的本地环境运行，CI 始终使用离线数据和 mock provider。

## 2. 公开字段最小化

公开样本只保留分析与复现需要的字段：

- 稳定编号；
- 平台类别和平台内顺序；
- 时间窗、粗主题和回复类型；
- 可选日期；
- 公开脱敏文本；
- 数据迁移状态。

公开证据保留父样本、文本、话题、机制、编码来源和纳入状态。来源链接、帖子标题、账号、UID 和平台内部定位字段不进入公开文件。

## 3. 文本脱敏

公开前扫描并处理：

- URL；
- 邮箱和手机号；
- @用户名；
- QQ、微信号及其他联系方式；
- 可能直接定位个人或原帖的字段。

脱敏只处理识别风险和格式噪声，不改写观点含义。公开数据更新时，URL 扫描、字段白名单和 manifest 哈希属于阻断检查。

## 4. 平台访问

真实采集需要遵守目标平台使用条款、访问频率限制和适用法律。采集实现必须包含限速、重试、缓存和停止条件。页面结构、接口参数和登录方式可能变化，历史实现不能被默认视为长期稳定工具。

## 5. 数据解释

公开文本来自社区讨论，不能据此推断个体身份、人格或临床状态。心理机制标签用于组织文本证据和形成研究问题，不构成心理诊断。等额抽样结果也不能直接解释为总体玩家意见比例。
