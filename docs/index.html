项目概览

一条从原始反馈延伸到产品假设的分析链路

项目同时保留样本层和证据层。样本层记录完整反馈，证据层将其中可单独判断的表达拆出并编码， 便于后续核对、统计和复盘。

01

数据基础

当前公开数据包含 若干 条样本，覆盖 数个 个平台，各平台样本量相近。

02

证据结构

每条证据都有稳定编号，并能关联到对应样本。证据数量可能多于样本数量，因为一条反馈可以包含多个独立观点。

03

当前产物

公开仓库提供脱敏数据、编码手册、评测方法、离线 Demo 与项目说明，方便查看方法和复现实例。

处理流程

六个阶段，逐步收束分析范围

  1. 01反馈整理

    统一字段并保留必要上下文。

  2. 02证据拆分

    切出可以单独判断的表达。

  3. 03主题编码

    识别反馈讨论的具体问题。

  4. 04机制编码

    记录解释框架与不确定项。

  5. 05洞察汇总

    按证据组合形成结构化观察。

  6. 06可靠性评测

    核对关联、覆盖、复现与边界。

离线 Demo

用少量脱敏反馈走通完整流程

Demo 使用确定性 mock provider，在本地生成证据、草稿洞察、产品假设和评测报告。默认不联网，也不调用外部模型。

  • 结果写入独立 artifacts 目录
  • 包含 JSON、Markdown 与 HTML 报告
