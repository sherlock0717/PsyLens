它解决什么问题

面对成百上千条社区抱怨，很难判断哪些是真问题、哪些结论靠得住。PsyLens 用普通语言回答四件事：

  • 高频抱怨究竟集中在哪些方面；
  • 这些反馈如何被拆成一条条可回溯的证据；
  • 分析用的标签是否稳定、有多少暂时判不准；
  • 由此得到的洞察和产品建议，能否回到最初的原始反馈。

当前案例与数据组成

案例整合多个社区平台的公开反馈。样本与每平台数量由页面数据动态载入：当前共 若干 条样本，覆盖 数个 个平台，每个平台 数量相近 条。数据分两层：

  • 样本层：一条完整的反馈原文；
  • 证据层：从原文中切出的、可单独判断的证据单元。

因此证据数量不等于样本数量——一条反馈可能切出多条证据，也可能一条都不构成独立证据。仓库中同时保留了早期的 legacy 结果与经过重建、带稳定编号的 v2 数据。

分析与评测流程

  1. 反馈
  2. 证据单元
  3. 主题与机制编码
  4. 结构化洞察
  5. 产品假设
  6. 可靠性评测

流程细节与字段定义见仓库 README 与方法文档。

离线 Demo

输入几条反馈，Demo 会离线生成证据单元、草稿结构化洞察、待验证产品假设和一份评测报告。默认不联网、不调用模型。

python tools/run_demo.py

方法与当前状态

  • B 站标签是离线规则基线提案：由关键词规则依据编码手册生成，用于演示评测流程，不是人工或模型语义复核；
  • 有两条极短证据来源不确定，仍待人工确认；
  • 产品建议是待验证假设，需要实验验证；
  • 仓库提供脱敏示例与公开分析数据；原始链接和内部迁移文件不作为展示页入口。
