## 2. 分项风险判定

| 项 | 风险等级 | 依据 |
| --- | --- | --- |
| URL 含用户 ID | low–medium | URL 含 BV 号 / 贴吧 tid / NGA tid，但指向**内容**而非用户主页；NGA 楼层锚点可间接定位到具体楼层作者 |
| 文本含昵称 | low | 正文未检出 @昵称；抓取中间产物有作者名，但公开整洁样本已丢弃 |
| 论坛楼层 / 账号线索 | **medium** | NGA URL 精确到 `#postcontentN`（楼层），结合帖子可定位具体发言者 |
| 视频 BV 号 | low | 指向公开视频，非个人隐私 |
| 个人主页 | low | 未检出 space.bilibili.com / home.php 个人主页链接 |
| 电话/邮箱/QQ/微信 | low | 均 0 命中 |
| 地理位置 | low | 未检出明显地理定位信息 |
| 敏感身份 | low | 未检出 |
| 粗口 | low–medium | 18 行；多为对**装备/英雄/机制**的一般性负面词（如「垃圾装备」「煞笔英雄」），非针对具名个人 |
| 潜在诽谤 / 定向攻击 | low | 未见针对具名真实个人的攻击；均为泛化吐槽 |
| 平台使用条款风险 | **requires_decision** | 三平台（B 站/NGA/贴吧）公开内容的抓取与**再分发**是否符合各平台条款，需用户评估 |
| 大规模重新分发风险 | **requires_decision** | 360 条含精确 URL 的公开评论再分发，属需用户决策事项 |

## 3. 页面 vs 下载数据

- **页面**：`docs/index.html` **不展示原始 URL**（符合任务口径）。
- **下载 CSV**：`input_feedback_phase2_multiplatform_clean.csv` 的 `url` 列**完整保留**精确来源链接（含 NGA 楼层锚点）。这是页面与下载数据之间的隐私暴露差。

## 4. 建议（本阶段不执行任何删除 / 改写）

1. **是否继续公开下载 CSV 及其 `url` 列**：`requires_decision`，交用户单独决定。
2. 若继续公开：建议在后续阶段对 `url` 做**降精度**（如去掉 NGA `#postcontentN` 楼层锚点、去掉 `?fr=` 等追踪参数），保留可核查性同时降低对具体楼层作者的定位。
3. 粗口 18 行：保留原句用于研究可追溯性；如面向对外展示可另做脱敏视图，不改历史文件。
4. 在 `REPRODUCIBILITY.md` / 数据说明中补充「再分发边界与平台条款」提示。

> 综合隐私风险：**medium**（主因是 NGA 楼层级 URL + 下载数据含精确链接），并含两项 `requires_decision`（平台条款、再分发）。无 `high` 项。
