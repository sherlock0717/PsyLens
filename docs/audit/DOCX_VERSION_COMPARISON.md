## 2. 内容差异

- **v4 是 v3 的扩展 / 超集**：v4 新增「代表性证据单元」示例、「6.4 证据、洞察与建议的对应关系」、行动建议的「落地路径」，以及「项目价值」章节；标题也更完整（如「六、结果分析、**代表性证据**与跨平台比较」「七、行动建议矩阵**与落地路径**」）。
- v3 独有段落多为被 v4 改写替换的旧版表述（同义但更简）。
- 两版结论方向一致（胜任受挫为主线、公平威胁为次级），无相互矛盾的数字或结论。
- v4 引入「中高」等新的置信描述词。

## 3. 元数据

- 两版 `docProps/core.xml` 的 created/modified 均为 `2013-12-23T23:15:00Z`，`creator=python-docx`，`revision=1`。
- 说明：这是 **python-docx 生成文档的占位时间戳**，**不是真实创作/修改时间**，因此**不能据元数据判断版本新旧或真实性**（与 AGENTS 口径一致：不倒推历史运行元数据）。
- 判断版本新旧只能依据内容完整度：v4 明显更完整。

## 4. 结论与建议（修正口径；本阶段不执行删除 / 重命名 / 改链接）

- **v4 是后续重写的优先底稿（preferred rewrite base），不是当前可直接发布的正式版。**
  - 理由：v4 内容为 v3 超集（段落 343>268、表 12>9），补齐了证据—洞察—建议对应关系与落地路径，可作为改写基础；
  - 但 v4 **仍继承当前的错误证据链（parent_id 错位）、人工复核口径（无复核记录却暗示已复核）与「三平台」机制结论**（证据层实际仅 NGA+Tieba）。因此**不能**据现状直接定为正式发布版。
- **数据与页面口径修复后**，应基于 v4 生成新的**无版本号**正式文件：`PsyLens_project_brief.docx`；随后再**归档 v3 与 v4**（移入 `docs/files/archive/` 或类似），不直接删除，保留版本痕迹。
- **当前不得改下载链接**：`docs/index.html` L55、`docs/index.md` L97 仍指向 v3；这属于待修复的不一致，但在数据修复前**保持现状**，待正式版生成后统一更新。
- **是否需要整合**：v3 独有 26 段经比对均为被 v4 改写的旧版同义句，无独有事实；整合到 v4 底稿即可。

> 元数据不可用于判断真实性（见 §3）。本阶段结论仅为设计依据；实际重命名 / 归档 / 改链接 / 生成正式版由后续阶段在数据修复后人工执行。
