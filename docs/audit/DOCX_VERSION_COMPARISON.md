## 2. 内容差异

- **v4 是 v3 的扩展 / 超集**：v4 新增「代表性证据单元」示例、「6.4 证据、洞察与建议的对应关系」、行动建议的「落地路径」，以及「项目价值」章节；标题也更完整（如「六、结果分析、**代表性证据**与跨平台比较」「七、行动建议矩阵**与落地路径**」）。
- v3 独有段落多为被 v4 改写替换的旧版表述（同义但更简）。
- 两版结论方向一致（胜任受挫为主线、公平威胁为次级），无相互矛盾的数字或结论。
- v4 引入「中高」等新的置信描述词。

## 3. 元数据

- 两版 `docProps/core.xml` 的 created/modified 均为 `2013-12-23T23:15:00Z`，`creator=python-docx`，`revision=1`。
- 说明：这是 **python-docx 生成文档的占位时间戳**，**不是真实创作/修改时间**，因此**不能据元数据判断版本新旧或真实性**（与 AGENTS 口径一致：不倒推历史运行元数据）。
- 判断版本新旧只能依据内容完整度：v4 明显更完整。

## 4. 结论与建议（本阶段不执行删除 / 重命名）

- **推荐正式版本：v4**。理由：内容为 v3 超集（段落 343>268、表格 12>9），补齐了证据—洞察—建议的对应关系与落地路径，图片与核心数字与 v3 一致且无冲突。
- **是否需要整合**：不需要额外整合；v4 已覆盖 v3 主要信息。仅需确认 v3 独有的 26 段中是否有 v4 未保留、但仍希望保留的表述（经比对均为被改写的旧版同义句，无独有事实）。
- **建议重命名**：可将 v4 重命名为 `PsyLens_project_brief.docx`（正式版）。
- **另一版处理**：v3 建议**归档**（移入 `docs/files/archive/` 或类似），不建议直接删除，保留版本痕迹。
- **需要更新的链接**：
  - `docs/index.html` L55：`files/PsyLens_enterprise_project_brief_v3.docx`（下载按钮）→ 应指向正式版。
  - `docs/index.md` L97：同样指向 v3 → 应更新。
  - `README.md` L72-73 与 `公开版文件清单.txt`：说明「同时保留 v3/v4，待确认」→ 确认后更新为单一正式版。
- ⚠️ 当前**页面下载链接指向 v3（较旧、较简版）**，而更完整的 v4 未被页面引用，属需要修复的不一致（非阻断，但建议尽快处理）。

> 本阶段结论仅为设计依据；实际重命名 / 归档 / 改链接由后续阶段人工执行。
