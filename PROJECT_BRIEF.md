> 本文件是 Markdown 版项目说明，供在 GitHub 上直接阅读。

---

## 1. 项目摘要

PsyLens 是一套围绕**公开游戏社区反馈**搭建的分析工作流。它的目标不是简单统计玩家「是否在抱怨不公平」，而是进一步判断：**高频争议究竟更稳定地指向哪一类体验问题与心理机制**，并把结论沿证据链回溯到玩家原始表达，最终收束为可讨论的行动建议。

当前公开案例聚焦《英雄联盟》海克斯大乱斗模式，整合 NGA、贴吧、B 站三平台公开反馈，形成 360 条整洁样本、697 个证据单元、19 条验证洞察与一份行动建议矩阵。结果显示，海克斯大乱斗的高频争议更稳定地指向**胜任受挫（competence frustration）**，公平威胁是重要但次级的补充机制。

---

## 2. 背景与问题

游戏社区反馈通常高噪声、高情绪、高重复：同一类抱怨会以完全不同的语气反复出现，也常常把「玩法看不懂」「匹配不公」「队友摆烂」混在一句话里。直接做情绪统计，容易得到「大家都在骂不公平」这类粗糙结论，既难以指导产品，也难以复核。

PsyLens 想回答的是更细的问题：当玩家在吵的时候，**他们更稳定地在表达哪一类未被满足的体验需求**？这类问题需要把表达拆细、给出机制解释，并保留可回溯的证据。

---

## 3. 案例范围

- **对象**：《英雄联盟》海克斯大乱斗模式。
- **来源**：NGA、贴吧、B 站的公开社区反馈。
- **样本**：360 条整洁样本，三平台各 120 条（公开案例中的平衡样本，非自然舆情分布）。
- **时间窗**：样本同时保留近期窗口与历史对照窗口（w1 / w2）。
- **边界**：探索性专题案例，不外推为平台全体玩家的普遍性结论。

## 5. 方法框架

```
Public Feedback → Clean Input → Evidence Unit → Mechanism Label → Validated Insight → Action Matrix
```

- **Clean Input**：预清洗 + AI 辅助精修 + 按平台平衡合并 + 字段标准化。
- **Evidence Unit**：把一条反馈拆成可独立判断的证据单元（一句吐槽常同时含主题归因、体验判断与行动线索）。
- **Mechanism Label**：为证据单元标注 surface_topic 与心理机制标签，并给出置信度。
- **Validated Insight**：按共现频率/强度收束成洞察，并标注是否需人工复核。
- **Action Matrix**：把结构化结果压缩为分层行动建议。

完整链路、每步产物与判断边界见 `METHODOLOGY.md`。

## 6. 核心发现

- **主线**：玩法/机制争议是主轴；去除 uncertain 后，胜任受挫的证据单元明显多于公平威胁。玩家更常表达的是「看不懂、打不顺、抓不住有效策略、努力与结果不匹配」，而非单纯抽象地指向「系统不公平」。
- **次级机制**：公平威胁存在且重要，但更集中在匹配、奖励与部分规则归因节点，不会盖过整张图。
- **放大器**：队友互动与社区摩擦更像放大器，会把已有挫败推成信任/沟通缺口与关系裂口。
- **平台差异**：NGA 玩法/机制讨论信号最强；贴吧补充队友互动与历史窗口表达；B 站补充视频语境下的英雄体验与互动。三平台表达风格不同，主线一致。

> 以上为对现有公开结果的转述，不新增结论。数字以 `docs/files/` 实际文件为准。

## 7. 行动建议结构

行动建议以分层矩阵形式给出（`docs/files/05_action_matrix.json`，字段为 safe / balanced / bold）：

- **safe / 稳妥**：降低玩法理解成本（版本/模式说明、示例内容）。
- **balanced / 平衡**：改善匹配体验与反馈透明度；为队友互动做减震。
- **bold / 进阶**：把模式运营升级到「修可理解性」，同时回应设计逻辑与常见误区。

> 该矩阵为 **AI 生成/辅助生成的产品假设**，使用时需配合人工判断，不代表任何企业内部决策。

---

## 8. 项目价值（能力体现）

同一套材料，可从多个岗位视角阅读，也对应不同的可迁移能力：

- **用户研究（UXR）**：把散乱抱怨翻译成可继续验证的研究问题，适合接访谈、问卷、playtest 假设——体现问题定义与证据化能力。
- **产品分析 / 模式策划**：把「先修哪里、先讲哪里、先碰哪个节点」排成优先级，避免一头扎进最吵但未必最核心的问题——体现结论优先级与决策支持能力。
- **社区反馈治理 / 发行**：识别最会放大情绪的互动点，判断何时该沟通、解释、减震——体现社区健康与反馈运营能力。
- **AI 辅助工作流**：在清洗、分类、洞察收束、建议生成中引入 AI，同时保留人工复核边界与可追溯证据——体现「AI + 人工」协作工作流的设计与治理能力。

## 9. 局限性

- 数据来自公开社区，不代表所有玩家。
- 三平台各 120 条为平衡样本，不等于自然舆情分布。
- 心理机制标签是解释性框架，不是临床测量。
- AI 辅助分类存在误差，需人工复核。
- 行动建议是基于公开反馈的产品假设，不代表 Riot 或任何企业的内部判断。
- 公开仓库不具备完整一键复现条件（见 `REPRODUCIBILITY.md`）。
